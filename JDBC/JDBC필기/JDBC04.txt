
 EXCEPTION -> 예외

 즉, 코드에 있어서 예외가 발생하게 되는것은 예외처리를 해주어야 함

 기존 코드에서보면 DAO 클래스에서 SQL 처리 시 발생하는 각종 예외에 대해
 DAO 에서 예외 에러 메세지를 출력하고 있음

 하지만 실제적으로 DAO에서 예외 발생한다 하여도 DAO에서 예외 메세지를
 보내고 끝내기만 하면 실제 DAO를 호출한 Service 혹은 Controller에서는
 해당 예외 상황을 알지 못한 상황이 됨

 보통 예외처리는 해당 메소드를 최초로 호출한 쪽에서 처리 하는 것이 맞음
 -> 즉, 전체 조회의 경우 컨트롤러에서 전체조회 서비스를 요청하였기 때문에
 예외가 발생하면 컨트롤러에게 예외 상황 대한 메세지가 발생하고
 처리할 수 있도록 해주는게 기본 비지니스 로직에 알맞음

 @ JTable
 -> JTable은 데이터를 보여주기 위한 뷰를 제공하는 테이블
 -> JTable에는 컬럼명과 데이터가 존재함
  ※ 컬럼명은 DB의 컬럼명과 같고, 데이터 DB의 row와 같음

  1. 컬렴명
  2. 데이터
  3. 컬럼명과 데이터를 담을 모델
  4. 모델을 담을 테이블
  5. 테이블의 정보가 많아지면 스크롤


  @ 스피너
  -> 스피너는 두 개의 화살표가 붙은 텍스트 필드
  -> 일반적으로 스피너의 텍스트 필드는 숫자를 포함하고 있고,
  사용자는 화살표를 클릭하여 값들을 증가시키거나 감소 시킬 수 있음
  -> 스피너를 이용하여 다양하게 사용할 수 있음
      (이 중 우리는 스피너를 이용하여 숫자를 증강 하는 형태로 사용 하겠음)

  ## 스피너 객체 생성 ##

  1. 스피너 모델 생성
  SpinerModel numberModel = new SpinerModel(0, -100, +100, 1); // 초기, 최대, 최소, 단계

  2. 스피너 생성
  JSpiner jsp = new JSpiner(numberModel);

  3. 스피너 사이즈 설정
  spinner.setSize(new Dimension(100, 30));
